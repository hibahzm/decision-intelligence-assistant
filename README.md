# Decision Intelligence Assistant

An AI‑powered customer support assistant that helps agents handle tickets faster and more consistently.  
It combines a **trained ML model for urgency classification** with a **RAG (Retrieval‑Augmented Generation) system** that suggests responses based on past successful support interactions.

---

## 🧠 How It Works

1. **User (support agent)** enters a customer tweet or query via the React frontend.
2. The backend:
   - Extracts 10 engineered features from the text (sentiment, keyword counts, punctuation, etc.).
   - Uses a pre‑trained **Random Forest classifier** to predict urgency: `urgent` or `normal`.
   - Embeds the query using `all‑MiniLM‑L6‑v2` (sentence‑transformer) and retrieves the most similar past tickets from **Qdrant** (vector database).
   - Calls **GPT‑4o‑mini** (OpenAI) to generate a helpful response, incorporating the retrieved past conversations as context.
3. The frontend displays the predicted urgency, the generated answer, and sources (similar tickets).

## 🚀 Getting the ML Model and Vector Data

The repository does **not** include pre‑trained models or vector data because they are specific to your dataset.

### Step A – Run the Companion Notebook in Google Colab

We provide a **Colab notebook** that:

1. Loads your customer tweet dataset (CSV).
2. Cleans text, builds conversations, and labels urgency using a rule‑based heuristic.
3. Engineers the 10 important features.
4. Trains a Random Forest classifier and saves `model.pkl`.
5. Generates embeddings (using `all‑MiniLM‑L6‑v2`) and stores them in a local Qdrant SQLite database (`qdrant_db/`).

After running the notebook, you will have:
- `model.pkl` (download it to `backend/ml/model.pkl`)
- `qdrant_db/` folder (on your Google Drive or local machine)

### Step B – Upload Vectors to Qdrant Cloud (Recommended)

For production or consistent deployment, use **Qdrant Cloud**:

1. Create a free cluster at [cloud.qdrant.io](https://cloud.qdrant.io).
2. Copy your cluster URL and API key.
3. Run the provided migration script (`backend/migrate_to_qdrant_cloud.py`) to upload vectors from your local `qdrant_db` to the cloud.

This step is **one‑time**. After completion, you can discard the local `qdrant_db` folder (or keep it as backup).

## 🛠️ Local Setup with Docker Compose

### 1. Clone the repository

```bash
git clone https://github.com/your-username/decision-intelligence-assistant.git
cd decision-intelligence-assistant
```

### 2. Place the trained model

```bash
mkdir -p backend/ml
cp /path/to/downloaded/model.pkl backend/ml/
```

### 3. Set environment variables

#### Backend secrets (`backend/.env`)

```ini
QDRANT_MODE=cloud
QDRANT_CLOUD_URL=https://your-cluster.cloud.qdrant.io
QDRANT_API_KEY=your-qdrant-api-key
OPENAI_API_KEY=sk-...
```

#### Root `.env` (optional, for Docker Compose variable substitution)

Create `.env` in the project root:

```ini
VITE_API_URL=http://localhost:8000
```

### 4. Build and run with Docker Compose

```bash
docker compose up --build
```

After startup, open:

- **Frontend**: http://localhost:3000
- **Backend API docs**: http://localhost:8000/docs


## 🧪 Testing the API

```bash
# Urgency prediction (ML model)
curl -X POST http://localhost:8000/api/predict \
  -H "Content-Type: application/json" \
  -d '{"text": "My phone is broken and I need a refund ASAP"}'

# RAG answer generation
curl -X POST http://localhost:8000/api/rag-answer \
  -H "Content-Type: application/json" \
  -d '{"query": "How to handle a delayed refund request?"}'
```

Interactive Swagger UI at http://localhost:8000/docs.
